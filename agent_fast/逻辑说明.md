# Fast 版逻辑说明

这套 `agent_fast` 是“跳过文生图”的实验路线。

原版逻辑是：

```text
A1 context -> 文生图首帧 -> 图生视频 prompt -> Seedance 图生视频
```

fast 版逻辑是：

```text
A1 context -> Seedance 文生视频 prompt -> Seedance 文生视频
```

## 为什么这样改

fast 版不再需要文生图中间层。A1 context 里已经有 background、lifestyle_theme、action_theme、mood_theme，足够作为文生视频 prompt 的事件种子。

也就是说，Fast Phase C 只读取：

```text
phase_a_contexts.jsonl
```

它不读取：

```text
文生图 prompt 文件
```

## 这一版少了什么

少了三步：

- 不生成文生图 prompt。
- 不跑 Replicate 文生图。
- 不生成首帧图。

因此 fast 版没有“首帧图”来锁住画面构图。

## 这一版多注意什么

因为没有首帧图，prompt 必须写清楚：

- 蘑菇TUTU在哪里。
- 周围有什么可互动道具。
- 它先做什么、接着做什么、最后停在什么状态。
- 镜头怎么稳定跟随。
- 光照和环境如何保持稳定。

也要明确：

- 角色参考图不是首帧，也不负责锁定场景构图。
- prompt 里不要写“以输入图片为第一帧”。
- prompt 里不要写“保持首帧构图”。
- prompt 里不要写 `4cm`、`微缩`、`微小体量`、`微小感`。

## 运行顺序

如果已经有 A1 context，可以直接跑 Fast Phase C：

```powershell
& 'F:\workspace\tutu内容\_tools\python311\python.exe' 'F:\workspace\tutu内容\agent_fast\phase_c_seedance_t2v_prompts.py' --limit 50
```

再 dry-run 看 Seedance payload：

```powershell
& 'F:\workspace\tutu内容\_tools\python311\python.exe' 'F:\workspace\tutu内容\agent_fast\phase_d_seedance_t2v_videos.py' --limit 1
```

确认没问题后提交：

```powershell
$env:ARK_API_KEY='你的 Ark API key'
& 'F:\workspace\tutu内容\_tools\python311\python.exe' 'F:\workspace\tutu内容\agent_fast\phase_d_seedance_t2v_videos.py' --limit 1 --execute --wait --download --force
```

## 推荐测试方式

先跑：

```powershell
--limit 3
```

看三件事：

- TUTU 外观是否稳定。
- 动作是否明显。
- 场景有没有乱。
