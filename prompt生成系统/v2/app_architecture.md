## 后端服务详解

### 1. 生活调度器（Daily Scheduler）

```
触发: 每天凌晨4:00 自动运行
      + 用户互动后实时追加

┌─────────────────────────────────────────────┐
│  输入                                        │
│  · 今天日期/星期/节气/节日                    │
│  · 用户GPS → 城市 → 天气API                  │
│  · 热点信号（来自热点转化器）                  │
│  · 用户最近互动（最新20条）                    │
│  · 秃秃过去7天生活日志                        │
│  · 秃秃人格设定                              │
├─────────────────────────────────────────────┤
│  LLM决策                                     │
│  · 生成3-5条今日事件（带时间戳）              │
│  · 每条事件的视频prompt + 心理活动            │
│  · 标记哪些受用户影响                         │
├─────────────────────────────────────────────┤
│  输出                                        │
│  · 视频prompt → Seedance生成队列             │
│  · 心理活动 → 存入数据库                      │
│  · 按时间戳定时发布到视频流                   │
└─────────────────────────────────────────────┘

发布策略:
  · 不是一次性全发，按事件时间戳陆续发布
  · 08:30的事件 → 08:30左右推送到用户视频流
  · 给用户"秃秃是活的"的感觉
```

### 2. 聊天服务（Chat Service）

```
用户发消息 → 实时处理

┌─────────────────────────────────────────────┐
│  用户输入                                    │
│  · 文字消息                                  │
│  · 图片（先过视觉模型提取描述）              │
├─────────────────────────────────────────────┤
│  LLM生成回复                                 │
│  上下文:                                     │
│  · 秃秃人格设定                              │
│  · 当前秃秃在干嘛（今日日程进度）             │
│  · 过去的对话记录（最近20条）                 │
│  · 秃秃的情绪状态                            │
│                                              │
│  输出:                                       │
│  · 文字回复（秃秃口吻，30-80字）             │
│  · 是否触发额外视频事件（bool）               │
│    → 如果是：生成1条回应视频加入今日日程      │
├─────────────────────────────────────────────┤
│  存储                                        │
│  · 用户消息 → user_memory                    │
│  · 图片描述 → user_memory                    │
│  · 影响后续日程生成                           │
└─────────────────────────────────────────────┘

触发额外视频的条件:
  · 用户分享了图片 → 秃秃做相关的事
  · 用户表达了情绪（累/开心/难过）→ 秃秃回应
  · 用户问秃秃在干嘛 → 发一条当前状态视频
  · 冷却: 每天最多触发2条额外视频
```

### 3. 热点信号转化器（Trend Skill）

```
触发: 每天早上7:00 自动运行

┌─────────────────────────────────────────────┐
│  Step 1: 抓取热点                            │
│  · 微博热搜 Top10                            │
│  · 知乎热榜 Top5                             │
│  · 天气预警                                  │
│  · 今日节日/节气/纪念日                       │
├─────────────────────────────────────────────┤
│  Step 2: LLM过滤+转化                        │
│                                              │
│  Prompt:                                     │
│  "你是秃秃的生活策划。以下是今日热点，        │
│   请筛选出秃秃能做的事（最多3条），            │
│   并转化为具体的生活场景信号。                 │
│                                              │
│   规则:                                      │
│   · 秃秃只有4cm高，在家/窗台/花盆活动        │
│   · 不涉及政治/负面/争议性话题                │
│   · 自然现象/季节变化/可爱话题优先            │
│   · 输出为具体场景，不是抽象概念"             │
│                                              │
│  示例:                                       │
│  热点"樱花季到了" →                          │
│  信号: "窗台花盆里的小花开了，秃秃凑近看"     │
│                                              │
│  热点"全国降温" →                            │
│  信号: "秃秃裹着纸巾缩在角落，冷得发抖"      │
│                                              │
│  热点"某明星离婚" → 过滤掉（不相关）          │
├─────────────────────────────────────────────┤
│  输出: 0-3条场景信号                          │
│  存入 daily_signals.json                     │
│  供生活调度器读取                              │
└─────────────────────────────────────────────┘
```

## API接口

### 视频流

```
GET /api/feed?before={timestamp}&limit=10
→ 返回视频列表（分页，按时间倒序）

GET /api/feed/today
→ 返回今日已发布的事件
```

### 聊天

```
POST /api/chat/send
  body: { "type": "text|image", "content": "...", "image_url": "..." }
→ 返回秃秃回复 + 是否触发视频

GET /api/chat/history?before={timestamp}&limit=20
→ 聊天记录
```

### 状态

```
GET /api/tutu/status
→ 返回秃秃当前状态（在干嘛/心情/今日进度）
```

## 定时任务

```
04:00  生活调度器 → 生成今日3-5条日程 → 提交Seedance
07:00  热点转化器 → 抓取热点 → 转化信号 → 可能追加1条日程
08:30  发布第1条视频（早晨）
~      按日程时间戳陆续发布
22:00  发布最后1条（晚安）
实时    用户聊天 → 可能追加1-2条视频
```

## 技术选型建议

```
前端: React Native / Flutter（跨端）
后端: Python FastAPI（与现有pipeline一致）
LLM:  Gemini API（已验证通）
视频: Seedance API（已验证通）
存储: SQLite（初期）→ PostgreSQL（扩展）
定时: APScheduler / cron
推送: APNs / FCM
```
