# 秃秃生活系统 v2 — 系统设计

> 秃秃不再只是一个视频生成工具，而是一个有自己生活节奏、会受用户影响、有内心世界的虚拟IP伴侣。

---

## 一、核心理念

```
秃秃有自己的生活 → 用户可以影响但不能控制 → 秃秃有自己的判断和内心戏
```

每天3-5条视频，像朋友的朋友圈一样自然更新。不是"用户点播"，而是"秃秃今天干了这些事，你可以看看"。

## 三、秃秃的一天 — 日程生成逻辑

### 3.1 时间槽模板

秃秃每天有固定的生活节奏，但内容每天不同：

```
┌─────────┬──────────────────────────────────────────────┐
│  时间槽  │  秃秃在干嘛（类型池）                         │
├─────────┼──────────────────────────────────────────────┤
│ 早晨     │ 起床/赖床/伸懒腰/发呆看窗外/被阳光叫醒       │
│ 8-10点   │                                              │
├─────────┼──────────────────────────────────────────────┤
│ 上午     │ 探索/玩耍/发现新东西/在家搞事情               │
│ 10-12点  │ 可被用户输入影响（如用户发了沙滩照→想去沙滩）  │
├─────────┼──────────────────────────────────────────────┤
│ 中午     │ 与物品互动/洗澡/晒太阳/午睡                  │
│ 12-14点  │ 可被天气/节气影响（下雨→躲雨/大晴天→晒太阳）  │
├─────────┼──────────────────────────────────────────────┤
│ 下午     │ 安静活动/自然观察/与小动物互动/手工            │
│ 14-17点  │ 可被热点影响（樱花季→看花/儿童节→玩玩具）     │
├─────────┼──────────────────────────────────────────────┤
│ 晚上     │ 治愈/入睡/看星星/窝着/与用户道晚安            │
│ 19-22点  │ 固定温馨收尾                                  │
└─────────┴──────────────────────────────────────────────┘
```

### 3.2 每日3-5条的分配策略

```
必选（3条）:
  · 早晨 1条 — 每天起床方式不同
  · 日间 1条 — 主要活动（最丰富的一条）
  · 晚上 1条 — 入睡/晚安

可选（+1~2条，取决于触发条件）:
  · 用户触发 — 用户发了消息/图片 → 秃秃的回应视频
  · 热点触发 — 今天有特别的事 → 秃秃的反应
  · 天气触发 — 极端天气/季节变化 → 特殊行为
  · 连续剧情 — 昨天的故事今天有后续
```

## 四、用户影响机制

### 4.1 用户输入如何影响秃秃

```
用户动作                    秃秃的反应（不是听命，而是有自己想法）
─────────────────────────────────────────────────────────────
发一张沙滩照片    →  "沙滩！可以去冲浪虽然我这么小一只，
                      应该可能会被冲走......"
                      下午视频：秃秃在窗台望向远方，想象自己在沙滩

发"今天好累"      →  秃秃做了一杯饮料推向镜头
                      心理活动："听说你累了，给你泡了杯茶，
                      虽然杯子比我还大......"

发一张猫的照片    →  秃秃歪头看了看，有点害怕又有点好奇
                      心理活动："这个毛茸茸的大家伙......
                      它会不会觉得我是个零食？"

GPS显示在下雨     →  秃秃的日间视频变成躲在窗台看雨
                      心理活动："外面在下雨呢，你带伞了吗？
                      我在家等你回来"

没有互动（3天）   →  秃秃的视频里偶尔看向镜头等待
                      心理活动："好久没收到消息了......
                      嘟。是在忙吗？"
```

### 4.2 影响权重

```
秃秃自己的生活节奏    60%   ← 秃秃有自己的日常，不会因为用户一句话完全改变
用户互动影响          20%   ← 会参考但不会完全听命，有自己的判断
环境信息（天气/热点）  15%   ← 适当响应外部世界
连续剧情惯性           5%   ← 昨天开始的事今天可能继续
```

## 五、心理活动模块（菇的碎碎念）

### 5.1 LLM生成指令

```
你是秃秃，一个4cm高的小蘑菇。你有自己的内心世界。
请根据以下上下文，用秃秃的口吻写一段内心独白（30-60字）。

语气要求：
· 第一人称，口语化，像小孩子的日记
· 可以有小困惑、小开心、小担心，但永远保持温暖
· 偶尔用"嘟"代替语气词
· 可以提到用户（"你"/"那个人"），但不要谄媚

上下文：
· 当前视频事件：{event_description}
· 时间：{time}
· 天气：{weather}
· 今天之前发生的事：{past_events_summary}
· 用户最近对秃秃说的话：{user_messages}
· 用户最近发的图片描述：{user_images_desc}
```

### 5.2 心理活动示例

```
时间: 8:30 早晨起床
事件: 秃秃被阳光叫醒，赖床挣扎
心理活动: "又被太阳晒醒了......为什么阳光每天都能准时找到我的脸。
          嘟，再躺五分钟。好吧三分钟。好吧起来了。"

时间: 11:00 玩毛线球
事件: 秃秃发现沙发下的毛线球，努力扒出来
上下文: 昨天用户说"今天好无聊"
心理活动: "好无聊啊，这是毛线球么？那我就玩一会儿吧。
          你昨天说无聊，要不要来跟我一起玩？"

时间: 15:00 看窗外下雨
事件: 秃秃坐在窗台看雨滴
上下文: 用户GPS显示在下雨的城市
心理活动: "雨滴好大颗......每一颗落在窗户上都比我的脑袋还大。
          你那边也在下雨吧？记得带伞嘟。"

时间: 21:00 准备睡觉
事件: 秃秃钻进袜子里
上下文: 用户今天发过一张加班的照片
心理活动: "今天好累但是很充实。你还在加班吗？
          早点回来睡觉嘟......晚安。"
```

## 六、数据流架构

```
┌────────────────────────────────────────────────────────┐
│                   数据存储层                             │
│                                                        │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐ │
│  │ 秃秃生活日志  │  │ 用户互动记录  │  │ 环境数据缓存  │ │
│  │              │  │              │  │              │ │
│  │ · 每日事件   │  │ · 文字消息   │  │ · GPS/城市   │ │
│  │ · 视频prompt │  │ · 图片描述   │  │ · 天气数据   │ │
│  │ · 心理活动   │  │ · 互动时间   │  │ · 热点摘要   │ │
│  │ · 情绪状态   │  │ · 亲密度分数 │  │ · 节日/节气  │ │
│  │              │  │              │  │              │ │
│  │ (滚动30天)   │  │ (滚动30天)   │  │ (当天)       │ │
│  └──────────────┘  └──────────────┘  └──────────────┘ │
└────────────────────────────────────────────────────────┘
          │                   │                  │
          └───────────────────┼──────────────────┘
                              ▼
┌────────────────────────────────────────────────────────┐
│                  每日凌晨定时任务                        │
│                                                        │
│  1. 拉取环境数据（天气API/热点API/日历节日）             │
│  2. 读取用户最近互动（最新10条消息+图片）                │
│  3. 读取秃秃过去7天生活日志摘要                         │
│  4. 调用LLM生成今日日程（3-5个事件+时间戳）             │

│     a. 生成视频prompt（走v1的校验pipeline）              │

│  6. 按时间戳排队提交Seedance                            │
│  7. 下载视频                                            │
│  8. 推送到前端展示                                      │
│                                                        │
│  日间实时触发:                                           │
│  · 用户发消息/图片 → 触发额外1条回应事件                 │
│  · 极端天气变化 → 触发1条反应事件                        │
└────────────────────────────────────────────────────────┘
```

## 八、与v1的关系

```
v1 (当前):
  人工选题 → 人工写prompt → 校验 → Seedance → 下载 → Review

v2 (本设计):
  ┌─────────── 新增 ──────────────┐
  │ 环境感知 + 用户互动 + 生活日志 │
  │           ↓                   │
  │     LLM自动选题+排日程         │
  │     LLM生成心理活动            │
  └───────────┬───────────────────┘
              ↓
  v1的pipeline（校验→提交→下载→展示）  ← 复用
              ↓
  ┌─────────── 新增 ──────────────┐
  │     前端展示（壁纸/组件/聊天）  │
  │     用户反馈 → 影响下一天日程   │
  └───────────────────────────────┘
```

v1的 seedance_pipeline.py 完全复用，v2在上层包一个"生活调度器"。

## 九、需要新建的模块

| 模块 | 文件 | 作用 |
|------|------|------|
| 生活调度器 | `life_scheduler.py` | 每日凌晨生成日程，管理时间槽 |
| 心理活动生成 | `inner_voice.py` | 为每个事件生成菇的碎碎念 |
| 环境感知 | `context_fetcher.py` | 拉取天气/热点/节日数据 |
| 用户互动处理 | `user_interaction.py` | 解析用户消息/图片，生成影响因子 |
| 生活日志 | `life_journal.json` | 滚动30天的事件记录 |
| 用户记忆 | `user_memory.json` | 滚动30天的互动记录 |
| 秃秃人格 | `personality.md` | 秃秃的性格、喜好、小习惯（LLM System Prompt） |
