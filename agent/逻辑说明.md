# 这套逻辑

现在这套流程已经从 prompt 到视频跑通了。

最简单的理解是：

**A1 负责想清楚方向，A2 写文生图 prompt，Phase B 生首帧，Phase C 写视频 prompt，Phase D 生视频并下载。**

也就是：

`A1 context -> A2 text_to_image_prompt -> 文生图首帧 -> Seedance prompt -> Seedance 视频`

如果想看 20 条完整例子，可以直接看：[20个完整流程示例.md](/f:/workspace/tutu内容/agent/20个完整流程示例.md)

## 为什么这样更简单

因为 A2 已经直接生成文生图 prompt 了，所以中间不用再多拆几层。

现在可以直接理解成：

1. A1 先把 daily、weather、background、lifestyle、action、mood 融合成一个 context
2. A2 直接读这个 context，生成一条文生图 prompt
3. 用这条 prompt 去生成首帧图
4. 等首帧图出来，再用 Phase C 系统提示词单独写 Seedance 图生视频 prompt
5. Phase D 把 prompt、首帧图、`reference.png` 组成 Ark payload，提交 Seedance 任务
6. 轮询任务状态，拿到 `video_url` 后下载 mp4

## A1 在干嘛

A1 就是“整理方向”。

它不直接写图，也不写视频。

它只是把几个参考模块融合一下：

- 时间氛围是什么
- 天气带来什么感觉
- 背景环境大概是什么
- 生活质感是什么
- 秃秃在做什么
- 秃秃是什么情绪

现在背景只用一个 `background` 字段，不再拆成 `background_env / background_topic / background_reason`。

季节也只保留季节和节气，不再带地区或城市。

这些模块不是素材库。

不是让 Gemini 从里面抽句子。

而是让 Gemini 理解方向，然后自己生成更自然的组合。

## A2 在干嘛

A2 现在就是核心层。

它直接生成：

```json
{
  "title": "枕边垫纸",
  "summary": "蘑菇TUTU把一小角纸巾垫到柔软枕头边缘，形成一个安静的生活画面。",
  "triggered_by": "action",
  "text_to_image_prompt": "蘑菇TUTU在柔软蓬松的枕头边缘，用两只小手把一小角纸巾垫到枕面凹陷处，安静明亮的室内生活氛围，画面干净通透，柔和的光束，奶油般柔和的散景"
}
```

这里真正要拿去文生图的是：

`text_to_image_prompt`

`title / summary / triggered_by` 只是保留给后面做记录和接 Seedance 用。

## A2 的重点规则

A2 写的是蘑菇TUTU文生图 prompt，所以重点是：

- 必须写 `蘑菇TUTU`
- 内部设定里它是微缩生物，但文生图 prompt 不要直接写 `4cm`、`微缩` 或任何说明它很小的语句
- 不要在文生图 prompt 里写任何说明它很小的语句
- 必须让它正在做一件具体小事，并且和身边物体互动
- 不要只写它站着、停着、观察、看着、被吸引、好奇靠近
- 不要总是让它穿衣服
- 不要总是让它表情夸张
- 可以是自然状态、恬静、无表情

同时有一些词不要写：

- 橙色伞盖
- 白点
- 米色身体
- 4cm
- 4cm 高的微缩有生命蘑菇
- 微缩有生命蘑菇
- 站在某处观察
- 被某物吸引
- 发光粒子
- 照亮的尘埃
- 漂浮的灰尘
- 圆形光斑

如果要写光，可以写：

- 柔和的光束
- 通透的光影
- 空气感
- 明亮均匀的光线

如果要写背景，可以写：

- 奶油般柔和的散景
- 朦胧的色块

## 后面怎么接 Seedance

不要在 A2 里直接写 Seedance 视频 prompt。

原因很简单：

**图还没出来。**

Seedance 是图生视频，所以后面应该等首帧图出来以后，再把：

- 首帧图
- A2 的 `text_to_image_prompt`
- `title`
- `summary`

一起给 Gemini，让它写 Seedance 的图生视频 prompt。

这样视频 prompt 才会贴着真实首帧图，不会凭空写动作。

现在这层系统提示词放在：

[phase_c_seedance_i2v_system_prompt.md](/f:/workspace/tutu内容/agent/phase_c_seedance_i2v_system_prompt.md)

它的意思就是：

- 真实首帧图优先
- A1/A2 只帮它理解“这张图原本想做什么”
- 不往画面里硬加不存在的东西
- 把蘑菇TUTU当前动作延续成一段连续小事件
- 按 15 秒视频来安排节奏，不要太短也不要写成长剧情
- 背景、构图、光照保持稳定，但 TUTU 自己要动得清楚、有幅度、看得见
- 自然写一点表情或眼神变化，不要套死模板
- 镜头稳定跟着看，不要拉近，不要乱切，也不要把重点写成“极轻微运镜”
- 少写“极轻微、微幅、小范围、几毫米、几乎不动、安静收住、轻轻停住”这类会让动作变僵的词
- 最后只输出一段中文视频 prompt

现在 Phase C 只给 Gemini 一张图：

- 每条内容自己的首帧图

`reference.png` 不参与 Phase C 写 prompt，它只在 Phase D 真正提交 Seedance 生视频时作为第二张参考图传给 Ark。

所以视频 prompt 的开头仍然写：

```text
以输入图片为第一帧，第二张图片为参考图，保持整体构图、透视关系与空间结构一致
```

## 当前最短流程

现在就记这一条：

```text
A1 融合方向
-> A2 生成文生图 prompt
-> 文生图生成首帧
-> 结合首帧图生成 Seedance 视频 prompt
-> 调 Seedance API
-> 下载 mp4
```

这更直接，也更适合你现在要先大批量生首帧图的阶段。

当前批量生成建议用 `gemini-2.5-flash`，配合 `--batch-size 25`，速度会比 3.1 Pro Preview 稳定很多。

## Replicate 生成图

现在补了一个执行层：

`phase_b_replicate_images.py`

它会读取 `phase_a_text_to_image_prompts.jsonl`，把每条 `text_to_image_prompt` 放进 Replicate 的 `prompt` 字段，创建 prediction，等待完成，并可选下载图片。

当前可用的文生图 deployment 是 `picaa/qwen-image-distilled-test-1109`。之前测试过 `picaa/qwen-image-edit-with-sam3`，它运行时会报 `未提供输入图片，无法生成内容`，所以不适合直接从 prompt 文生图。

```powershell
$env:REPLICATE_API_TOKEN='你的 Replicate token'
& 'F:\workspace\tutu内容\_tools\python311\python.exe' 'F:\workspace\tutu内容\agent\phase_b_replicate_images.py' --input-jsonl 'F:\workspace\tutu内容\agent\outputs\gemini_25_flash_50_b25\phase_a_text_to_image_prompts.jsonl' --deployment 'picaa/qwen-image-distilled-test-1109' --limit 1 --seed 1000 --download
```

Seedance 生视频任务脚本是：

`phase_d_seedance_videos.py`

先 dry-run 看 payload：

