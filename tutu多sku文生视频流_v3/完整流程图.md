# TUTU Multi-SKU 5s Pipeline 完整流程图

> 本文档展示 `tutu多sku文生视频流/` 从 Phase A 到 Phase D 的完整数据流、文件 I/O 和 system prompt 注入过程。
>
> 所有流程图用 mermaid 语法写，支持 GitHub / VS Code / Obsidian 等主流 markdown 渲染器。每个图下面附一段文字说明。

---

## 1. 总览图：四个 Phase 的串联

```mermaid
flowchart LR
    subgraph A[Phase A: A1 Context]
        A1[按 slot × weather<br/>生成 blueprint]
        A2[Gemini 融合填<br/>background/lifestyle/<br/>action/mood]
        A1 --> A2
    end

    subgraph B[Phase B: Multi-SKU Blueprint]
        B1[遍历每条 context]
        B2[随机抽 1/7 SKU<br/>resolve 3 张图路径<br/>派生标题]
        B1 --> B2
    end

    subgraph C[Phase C: T2V Prompt]
        C1[遍历每条 blueprint]
        C2[Gemini 生成<br/>5 秒 T2V prompt]
        C3[sanitize + validate<br/>自动重试 2 次]
        C1 --> C2 --> C3
    end

    subgraph D[Phase D: Ark Seedance]
        D1[组 payload:<br/>prompt + 3 张参考图]
        D2[POST Ark]
        D3[轮询任务状态]
        D4[下载 mp4]
        D1 --> D2 --> D3 --> D4
    end

    A -->|phase_a_contexts.jsonl| B
    B -->|phase_b_multi_sku_blueprints.jsonl| C
    C -->|phase_c_multi_sku_t2v_prompts.jsonl| D
    D -->|mp4 文件| E[outputs/multi_sku_t2v_videos/videos/]

    style A fill:#e3f2fd
    style B fill:#e8f5e9
    style C fill:#fff3e0
    style D fill:#f3e5f5
```

**读图要点**：

- **Phase A** 只负责规划创意上下文，调 Gemini
- **Phase B** 做确定性准备工作：挑 SKU、查路径、写标题，**不调 LLM**
- **Phase C** 把 blueprint 改写成实际喂给 Seedance 的中文 prompt，调 Gemini
- **Phase D** 不调 LLM，只做 Ark API 调用和 mp4 下载
- 四阶段之间通过 jsonl 文件解耦，任一阶段都可以单独重跑

**拆成 A/B/C/D 的好处**：

- 换 system prompt → 只重跑 Phase C，SKU 分配保持稳定可比
- 换 SKU 随机种子 → 只重跑 Phase B，context 不动
- 添加新 SKU → 只重跑 Phase B，前后都不用动
- Ark API 提交失败 → 只重跑 Phase D

## 2. Phase A 详细流程（含 system prompt 注入）

```mermaid
flowchart TD
    Start[phase_a.py --count 50]

    Start --> Load1[读 ip_data/personality.md]
    Start --> Load2[读 ip_data/ip-constitution.md]
    Start --> Load3[读 phase_a_system_prompt.md<br/>含 personality/constitution 占位符]
    Start --> Load4[读 phase_a_pool_subprompts.md<br/>解析成 7 池子 dict]
    Start --> Load5[读 ip_data/daily_signals.json<br/>热点话题池]

    Load4 --> Bprint[simulate_fused_context_blueprints<br/>按 slot × weather 组合 50 个 blueprint<br/>每个带 reference_hints 字典]
    Load5 --> Bprint

    Bprint --> Chunk[按 batch_size=10 切块]

    Load1 --> Render[替换占位符:<br/>personality → personality.md 全文<br/>constitution → ip-constitution.md 全文]
    Load2 --> Render
    Load3 --> Render

    Render --> SysPrompt[完整 system prompt]
    Chunk --> UserPrompt[user prompt = blueprint batch JSON]

    SysPrompt --> Gemini[调 Gemini 2.0 Flash]
    UserPrompt --> Gemini

    Gemini --> Parse[parse_json_block<br/>提取 JSON 数组]
    Parse --> Merge[每条 blueprint 填上<br/>background/lifestyle_theme/<br/>action_theme/mood_theme]
    Merge --> Validate[validate_context<br/>4 个字段都不能为空]
    Validate -->|失败| Fallback[build_context_fallback<br/>用 slot 默认值兜底]
    Validate -->|通过| Out
    Fallback --> Out

    Out[outputs/phase_a/TIMESTAMP/<br/>phase_a_contexts.jsonl<br/>manifest.json]

    style Start fill:#e3f2fd
    style SysPrompt fill:#c8e6c9
    style Gemini fill:#ffecb3
    style Out fill:#f8bbd0
```

**关键说明**：

1. **四份 md/json 文件一起组装 system prompt**：
   - `phase_a_system_prompt.md` 是模板（带两个 `{...}` 占位符）
   - 运行时替换 `{personality}` 和 `{constitution}` 成 IP 数据的完整内容
   - `phase_a_pool_subprompts.md` 不进 system prompt，而是作为 `reference_hints` 字典附在每条 blueprint 上
2. **blueprint 结构**：每条已有 `slot` / `weather` / `season`，**留空** `background` / `lifestyle_theme` / `action_theme` / `mood_theme` 四字段等 Gemini 来填
3. **batch 批处理**：50 条按 10 条一批调 Gemini
4. **Fallback 容错**：如果 Gemini 返回某条 context 缺字段，按 slot 默认值兜底

## 3. Phase B 详细流程（随机 SKU + 参考图路径 + 标题）

```mermaid
flowchart TD
    Start[phase_b_multi_sku_blueprints.py]

    Start --> Resolve[resolve_default_contexts:<br/>自动 glob outputs/phase_a/*/<br/>找最近的 phase_a_contexts.jsonl]
    Start --> RNG[random.Random seed=20260424]
    Start --> ImgCheck[检查 sku/hand_foot.jpg<br/>和 sku/mouth.jpg 存在]

    Resolve --> Loop{遍历每条 context}

    Loop --> Pick[pick_sku_index<br/>rng.randint 1-7]
    RNG --> Pick
    Pick --> ResolveSku[resolve_sku_path<br/>sku/i.png]

    Loop --> Title[title_from_context<br/>从 action_theme 派生 16 字标题]

    ResolveSku --> Build[build_blueprint]
    Title --> Build

    Build --> Record[blueprint 记录:<br/>index / context_id / title<br/>sku_index / 3 张图路径<br/>context 子字段passthrough]

    Record --> Loop

    Loop -->|全部完成| Out[outputs/multi_sku_blueprints/<br/>phase_b_multi_sku_blueprints.jsonl<br/>phase_b_multi_sku_blueprints.md]

    style Start fill:#e8f5e9
    style Pick fill:#fff9c4
    style Out fill:#f8bbd0
```

**关键说明**：

1. **完全不调 LLM**，只做确定性处理，通常几秒跑完
2. **SKU 随机时机**：在 Phase B 里随机（不再是 Phase C），写进 JSONL 的 `sku_index` 字段。`--seed` 控制复现
3. **三张参考图路径都在这一步定好**：`sku_image_path` / `hand_foot_image_path` / `mouth_image_path` 全部 resolve 成绝对路径并检查文件存在，避免 Phase D 提交时才发现路径错
4. **context 字段 passthrough**：把 A1 context 的 10 个字段嵌套在 blueprint 的 `context` 字段里，Phase C 拿到 blueprint 就有完整的上下文，不用再反查 A1 context 文件
5. **标题派生**：用 `action_theme` 前 16 个字符，剥去标点作为 `title`，用于后续视频文件名

## 4. Phase C 详细流程（blueprint → 5s T2V prompt）

```mermaid
flowchart TD
    Start[phase_c_multi_sku_t2v_prompts.py --limit 10]

    Start --> Resolve[resolve_default_blueprints:<br/>自动 glob outputs/multi_sku_blueprints/<br/>找最近的 phase_b_multi_sku_blueprints.jsonl]
    Start --> SysLoad[读 phase_c_multi_sku_t2v_system_prompt.md<br/>整份即 system prompt]

    Resolve --> Loop{遍历每条 blueprint}

    Loop --> UPrompt[build_user_prompt<br/>blueprint 里的 sku_index + context<br/>打包成 JSON]
    SysLoad --> Gemini[调 Gemini 3.1 Pro Preview]
    UPrompt --> Gemini

    Gemini --> Strip[strip_code_fence<br/>剥去 markdown 代码块]
    Strip --> Sanitize[sanitize_prompt<br/>删除:<br/>4cm、微缩尺度、不要超过画面 X<br/>分镜、切到 POV、镜头切换<br/>0-1.5s、第 3 秒等时间码]
    Sanitize --> Validate[validate_prompt_shape<br/>检查:<br/>以'图片1是蘑菇TUTU的四视图'开头<br/>含图片1/2/3<br/>含 风格:/镜头:/场景:/配乐/音效:/约束: 五段]

    Validate -->|失败| Retry{attempt<=2?}
    Retry -->|是| Gemini
    Retry -->|否| Fail[抛异常]

    Validate -->|通过| Write[写入 phase_c_multi_sku_t2v_prompts.jsonl<br/>带上 blueprint 的 sku_index /<br/>三张图路径 / title]
    Write --> Loop

    Loop -->|全部完成| Out[outputs/multi_sku_t2v_prompts/<br/>phase_c_multi_sku_t2v_prompts.jsonl<br/>phase_c_multi_sku_t2v_prompts.md]

    style Start fill:#fff3e0
    style SysLoad fill:#c8e6c9
    style Gemini fill:#ffecb3
    style Sanitize fill:#ffcdd2
    style Validate fill:#ffcdd2
    style Out fill:#f8bbd0
```

**关键说明**：

1. **输入是 Phase B 的 blueprint**（不是 Phase A 的 A1 context）：blueprint 里已经有 `sku_index` 和 `context`，Phase C 不再负责随机或查路径
2. **blueprint 文件自动发现**：默认不用传 `--blueprints-jsonl`，脚本会 glob 找最新的 Phase B 产物
3. **只做一件事：调 Gemini**。system prompt + blueprint 打包的 user prompt → T2V prompt
4. **双层质量闸门**：
   - `sanitize_prompt` 硬过滤违规词（抽象尺度、切镜语言、时间码）
   - `validate_prompt_shape` 检查结构（首段开头、图片引用、五段标签）
   - 任一失败 → 自动重试最多 2 次
5. **三张参考图路径从 blueprint 透传**到 Phase C 的 JSONL，Phase D 直接用

## 5. Phase D 详细流程（组 payload + Ark 调用 + 下载）

```mermaid
flowchart TD
    Start[phase_d_multi_sku_t2v_videos.py<br/>--limit 3 --execute --wait --download]

    Start --> Key[检查 ARK_API_KEY]
    Start --> Load[读 phase_c_multi_sku_t2v_prompts.jsonl]
    Start --> HF[hand_foot.jpg 本地 → data URL]
    Start --> MO[mouth.jpg 本地 → data URL]

    Load --> Loop{遍历每条 prompt 记录}

    Loop --> SkuURL[resolve_character_image<br/>sku_image_path → data URL]
    SkuURL --> Payload[build_payload:<br/>model=doubao-seedance-2-0-260128<br/>duration=5, ratio=9:16<br/>content = text + 3 × image_url<br/>每张 role=reference_image]
    HF --> Payload
    MO --> Payload

    Payload --> Dump[写 payloads/0000N.json]

    Dump --> Mode{--execute?}
    Mode -->|否 dry-run| Skip[状态标 dry_run]
    Mode -->|是| Post[POST Ark Seedance<br/>/api/v3/contents/generations/tasks]

    Post --> TaskId[拿 task_id]
    TaskId --> WaitMode{--wait?}

    WaitMode -->|否| WriteTask
    WaitMode -->|是| Poll[GET task status<br/>每 10 秒一次]
    Poll --> Check{status?}
    Check -->|running| Poll
    Check -->|succeeded| GetURL[拿 video_url]
    Check -->|failed/cancelled| WriteTask

    GetURL --> DLMode{--download?}
    DLMode -->|是| Download[下载 mp4<br/>videos/0000N_skuK_TITLE.mp4]
    DLMode -->|否| WriteTask
    Download --> WriteTask

    Skip --> WriteTask
    WriteTask[写 multi_sku_t2v_tasks.jsonl<br/>含 task_id / status /<br/>video_url / download_path]
    WriteTask --> Loop

    Loop -->|全部完成| Done[完成]

    style Start fill:#f3e5f5
    style Payload fill:#c5cae9
    style Post fill:#ffccbc
    style Poll fill:#fff9c4
    style Download fill:#b2dfdb
```

**关键说明**：

1. **无 LLM 调用**：Phase D 全程不调 Gemini，只和 Ark Seedance API 交互
2. **三张图片一起传**：`build_payload` 把 SKU 四视图 + 手脚图 + 嘴巴图三张都作为 `reference_image` 放进 content 数组
3. **本地图片 → data URL**：默认把本地 jpg/png 转成 `data:image/...;base64,...` 直接放进 `image_url.url`。如果 Ark 项目不支持 data URL，用 `--hand-foot-image-url / --mouth-image-url` 传公网 URL 覆盖
4. **三级开关**：
   - 不加 `--execute` → 只生成 payload JSON，不真提交（dry-run）
   - 加 `--execute` 但不加 `--wait` → 只创建任务，不等结果
   - 加 `--execute --wait --download` → 创建、轮询到结束、下载 mp4
5. **幂等**：任务状态落 `multi_sku_t2v_tasks.jsonl`，再次运行默认跳过已有 index。加 `--force` 重新跑

```mermaid

        B2 -.填进 reference_hints.-> B1

    end

    Blue --> PC_USER[Phase C user prompt:<br/>blueprint 的 sku_index + context JSON]
    PC_SYS --> GeminiC[Gemini Phase C]
    PC_USER --> GeminiC

    style PA_SYS fill:#c8e6c9
    style PC_SYS fill:#c8e6c9

    style GeminiC fill:#ffecb3
    style PhaseB fill:#e8f5e9
    style PhaseD fill:#f3e5f5
```

**关键说明**：

- **Phase A 的 system prompt** = `phase_a_system_prompt.md` + `{personality}` 替换 + `{constitution}` 替换
- **Phase A 的 user prompt** = blueprint JSON，其中 `reference_hints` 字典的 7 个键来自 `phase_a_pool_subprompts.md`
- **Phase B 没 system prompt**：只是 Python 代码做随机选择 + 路径解析
- **Phase C 的 system prompt** = `phase_c_multi_sku_t2v_system_prompt.md` 整份（不替换任何占位符）
- **Phase C 的 user prompt** = "请改写这条 blueprint" + blueprint 的 sku_index + context JSON
- **Phase D 没 system prompt**：只提交到 Ark

## 8. 数据字段跨 Phase 传递表

| 字段 | Phase A 产出 | Phase B 消费 | Phase B 产出 | Phase C 消费 | Phase C 产出 | Phase D 消费 | Phase D 产出 |
|------|-------------|-------------|-------------|-------------|-------------|-------------|-------------|
| `context_id` | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| `slot / weather / season` | ✓ | → 存进 context | ✓（嵌在 context 子字段） | ✓（融入 prompt 风格段） | - | - | - |
| `background` | ✓ | → 存进 context | ✓ | ✓ | - | - | - |
| `action_theme` | ✓ | → 存进 context、派生 title | ✓ | ✓（扩展成 5s 动作链） | - | - | - |
| `mood_theme` | ✓ | → 存进 context | ✓ | ✓（融入表情描述） | - | - | - |
| `title` | - | 从 action_theme 派生 | ✓ | ✓ 透传 | ✓ | ✓（文件名） | ✓ |
