# 这套逻辑

现在这套流程已经从 prompt 到视频跑通了。

最简单的理解是：

**A1 负责想清楚方向，A2 写文生图 prompt，Phase B 生首帧，Phase C 写视频 prompt，Phase D 生视频并下载。**

也就是：

`A1 context -> A2 text_to_image_prompt -> 文生图首帧 -> Seedance prompt -> Seedance 视频`

如果想看 20 条完整例子，可以直接看：[20个完整流程示例.md](/f:/workspace/tutu内容/agent_nano/20个完整流程示例.md)

## 为什么这样更简单

因为 A2 已经直接生成文生图 prompt 了，所以中间不用再多拆几层。

现在可以直接理解成：

1. A1 先把 daily、weather、background、lifestyle、action、mood 融合成一个 context
2. A2 直接读这个 context，生成一条文生图 prompt
3. 用这条 prompt 去生成首帧图
4. 等首帧图出来，再用 Phase C 系统提示词单独写 Seedance 图生视频 prompt
5. Phase D 把 prompt、首帧图、`reference.png` 组成 Ark payload，提交 Seedance 任务
6. 轮询任务状态，拿到 `video_url` 后下载 mp4

## A1 在干嘛

A1 就是“整理方向”。

它不直接写图，也不写视频。

它只是把几个参考模块融合一下：

- 时间氛围是什么
- 天气带来什么感觉
- 背景环境大概是什么
- 生活质感是什么
- 秃秃在做什么
- 秃秃是什么情绪

现在背景只用一个 `background` 字段，不再拆成 `background_env / background_topic / background_reason`。

季节也只保留季节和节气，不再带地区或城市。

这些模块不是素材库。

不是让 Gemini 从里面抽句子。

而是让 Gemini 理解方向，然后自己生成更自然的组合。

## A2 在干嘛

A2 现在就是核心层。

它直接生成：

```json
{
  "title": "枕边垫纸",
  "summary": "蘑菇TUTU把一小角纸巾垫到柔软枕头边缘，形成一个安静的生活画面。",
  "triggered_by": "action",
  "text_to_image_prompt": "蘑菇TUTU在柔软蓬松的枕头边缘，用两只小手把一小角纸巾垫到枕面凹陷处，安静明亮的室内生活氛围，画面干净通透，柔和的光束，奶油般柔和的散景"
}
```

这里真正要拿去文生图的是：

`text_to_image_prompt`

`title / summary / triggered_by` 只是保留给后面做记录和接 Seedance 用。

## A2 的重点规则

A2 写的是蘑菇TUTU文生图 prompt，所以重点是：

- 必须写 `蘑菇TUTU`
- 内部设定里它是微缩生物，但文生图 prompt 不要直接写 `4cm`、`微缩` 或任何说明它很小的语句
- 不要在文生图 prompt 里写任何说明它很小的语句
- 必须让它正在做一件具体小事，并且和身边物体互动
- 不要只写它站着、停着、观察、看着、被吸引、好奇靠近
- 不要总是让它穿衣服
- 不要总是让它表情夸张
- 可以是自然状态、恬静、无表情

同时有一些词不要写：

- 橙色伞盖
- 白点
- 米色身体
- 4cm
- 4cm 高的微缩有生命蘑菇
- 微缩有生命蘑菇
- 站在某处观察
- 被某物吸引
- 发光粒子
- 照亮的尘埃
- 漂浮的灰尘
- 圆形光斑

如果要写光，可以写：

- 柔和的光束
- 通透的光影
- 空气感
- 明亮均匀的光线

如果要写背景，可以写：

- 奶油般柔和的散景
- 朦胧的色块

## 后面怎么接 Seedance

不要在 A2 里直接写 Seedance 视频 prompt。

原因很简单：

**图还没出来。**

Seedance 是图生视频，所以后面应该等首帧图出来以后，再把：

- 首帧图
- A2 的 `text_to_image_prompt`
- `title`
- `summary`

一起给 Gemini，让它写 Seedance 的图生视频 prompt。

这样视频 prompt 才会贴着真实首帧图，不会凭空写动作。

现在这层系统提示词放在：

[phase_c_seedance_i2v_system_prompt.md](/f:/workspace/tutu内容/agent_nano/phase_c_seedance_i2v_system_prompt.md)

它的意思就是：

- 真实首帧图优先
- A1/A2 只帮它理解“这张图原本想做什么”
- 不往画面里硬加不存在的东西
- A2 的动作只作为视频事件的起点、动机或主体线索，不要把整段视频限制成重复同一个动作
- 不要让 TUTU 只围绕起始物体单纯移动、推拉或观察
- 后续动作要从首帧真实可见的物体、边界、材质、光线、空间层次和环境关系继续发散
- 视频要形成“起始动作 → 情境展开 → 多步互动 → 明确结果”的连续小故事
- 但丰富故事不等于硬加新场景、新角色或首帧不存在的重要道具
- 按 15 秒视频来安排节奏，不要太短也不要写成长剧情
- 每条视频 prompt 至少有 2-3 个清楚可见的动作阶段，不能只有一次触碰或一次观察
- 动作里要能看出 TUTU 的身体重心变化，比如前倾、后仰、蹲下、撑住、踮脚、转身、扶住物体、后退调整、再次发力
- 和首帧里真实可见的物体互动时，物体要有合理反馈，比如滑动、晃动、翻起、轻微旋转、弹回或停住
- 背景、构图、光照保持稳定，但 TUTU 自己要动得清楚、有幅度、看得见
- 表情或眼神要随动作推进变化，不要套死模板
- 镜头不用一直固定稳定看，必须明确写出镜头过程，不能只在结尾补一句“镜头稳定跟随”。可以根据动作阶段自然切换、变换景别和平滑跟随，比如中景看动作、侧向看发力、俯视看物体反馈、低角度跟随移动、再回到 TUTU 完成动作；最好写成“镜头先……随后切到……接着……最后……”。但切换必须顺着动作发生，不要跳出首帧空间，不要快速乱切，不要过度拉近，也不要把重点写成“极轻微运镜”
- 少写“极轻微、微幅、小范围、几毫米、几乎不动、安静收住、轻轻停住、轻轻、缓慢、安静、微微、小心翼翼”这类会让动作变僵的词
- 最后只输出一段中文视频 prompt

现在 Phase C 只给 Gemini 一张图：

- 每条内容自己的首帧图

`reference.png` 不参与 Phase C 写 prompt，它只在 Phase D 真正提交 Seedance 生视频时作为第二张参考图传给 Ark。

所以视频 prompt 的开头仍然写：

```text
以输入图片为第一帧，第二张图片为参考图，保持整体构图、透视关系与空间结构一致
```

## 当前最短流程

现在就记这一条：

```text
A1 融合方向
-> A2 生成文生图 prompt
-> 文生图生成首帧
-> 结合首帧图生成 Seedance 视频 prompt
-> 调 Seedance API
-> 下载 mp4
```

这更直接，也更适合你现在要先大批量生首帧图的阶段。

当前批量生成建议用 `gemini-2.5-flash`，配合 `--batch-size 25`，速度会比 3.1 Pro Preview 稳定很多。

## Nano Banana 生成图

现在补了一个执行层：

`phase_b_nano_banana_images.py`

它会读取 `phase_a_text_to_image_prompts.jsonl`，把每条 `text_to_image_prompt`、`agent_nano/reference.png`，以及“直接生成一张 9:16 竖版首帧图片、且小蘑菇TUTU的画面占比不得超过整体画面大小的五分之一、场景中不要出现尘埃，不要出现任何发光的粒子状，光线要柔和，不要有光点或圆形光斑，小蘑菇TUTU位置要符合物理规律，禁止悬浮和物理穿模”的指令一起发给 Gemini Nano Banana 文生图模型，并在请求里设置 `imageConfig.aspectRatio = 9:16`，保存首帧图，并写出 `nano_banana_predictions.jsonl`。

默认模型是 `gemini-2.5-flash-image`。如果模型名变化，可以用 `--model` 或环境变量 `GEMINI_IMAGE_MODEL` 覆盖。

这里的 `reference.png` 只用于约束蘑菇TUTU的角色外观和整体风格，不照搬参考图背景。后面 Phase D 仍然会再次把 `reference.png` 作为第二张参考图传给 Seedance，用来稳定视频阶段的角色外观。

```powershell
$env:GEMINI_API_KEY='你的 Gemini key'
& 'F:\workspace\tutu内容\_tools\python311\python.exe' 'F:\workspace\tutu内容\agent_nano\phase_b_nano_banana_images.py' --input-jsonl 'F:\workspace\tutu内容\agent_nano\outputs\gemini_25_flash_50_b25\phase_a_text_to_image_prompts.jsonl' --output-dir 'F:\workspace\tutu内容\agent_nano\outputs\nano_banana_images' --limit 1 --force
```
